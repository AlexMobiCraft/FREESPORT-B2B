# Контракты API (API Contracts)

Платформа предоставляет REST API для взаимодействия фронтенда и внешних систем. Все эндпоинты имеют префикс `/api/v1/`.

## 🔐 Аутентификация (`/auth/`)

| Метод | Эндпоинт          | Описание                                              |
| :---- | :---------------- | :---------------------------------------------------- |
| POST  | `/auth/register/` | Регистрация (B2C — сразу, B2B — требует верификации). |
| POST  | `/auth/login/`    | Получение Access/Refresh токенов.                     |
| POST  | `/auth/refresh/`  | Обновление Access токена (с ротацией Refresh).        |
| POST  | `/auth/logout/`   | Инвалидация токена (Blacklist).                       |

---

## 🛍️ Каталог товаров (`/products/`)

| Метод | Эндпоинт            | Описание                                                      |
| :---- | :------------------ | :------------------------------------------------------------ |
| GET   | `/products/`        | Список товаров с фильтрацией (категория, бренд, цена, флаги). |
| GET   | `/products/{slug}/` | Детальная информация о товаре, включая варианты (SKU).        |
| GET   | `/products/search/` | Полнотекстовый поиск (PostgreSQL FTS).                        |
| GET   | `/categories/`      | Дерево категорий.                                             |

---

## 🛒 Корзина и Заказы (`/cart/`, `/orders/`)

| Метод | Эндпоинт        | Описание                                        |
| :---- | :-------------- | :---------------------------------------------- |
| GET   | `/cart/`        | Содержимое текущей корзины (по JWT или сессии). |
| POST  | `/cart/add/`    | Добавление `variant_id` в корзину.              |
| POST  | `/orders/`      | Создание заказа из корзины (транзакционно).     |
| GET   | `/orders/{id}/` | Детали заказа и статус оплаты.                  |

---

## ⚙️ Особенности реализации (Frontend)

- **api-client.ts**: Базовый клиент на Axios.
- **SSR Handling**: Фронтенд определяет окружение (Server/Client) и использует соответствующий `baseURL`.
- **Error Handling**: Реализован механизм автоматического повтора запросов (`retry`) при 500/503 ошибках.
