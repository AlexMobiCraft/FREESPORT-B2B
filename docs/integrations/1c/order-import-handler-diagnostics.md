# Диагностика загрузки Организации и Склада в 1С

Цель: определить, почему при загрузке заказа в 1С поля документа `Организация` и `Склад` остаются пустыми, хотя сайт передает эти значения в XML.

## Что подтверждено на стороне сайта

На локальном Docker с текущей версией backend XML экспорта содержит значения:

- верхнеуровневые теги документа: `Организация`, `Склад`;
- дублирующие реквизиты в `ЗначенияРеквизитов`: `Организация`, `Склад`, `Соглашение`.

Пример диагностического XML: `docs/integrations/1c/samples/order-export-org-warehouse-diagnostic.xml`.

Фактический локальный audit XML: `backend/var/1c_exchange/logs/20260422_171023_orders.xml`.

## Итог локальной проверки `test_base`

Проверка выполнена на локальной серверной базе 1С `127.0.0.1\test_base`.

Конфигурация и расширение для дальнейшего анализа выгружены в локальную папку:

```text
C:\Users\1\Documents\test_base_dump_20260423_051033
```

Ключевой обработчик загрузки документов находится в выгруженном расширении:

```text
ОбменСБитриксУправлениеСайтомУТ\CommonModules\БУС_ЗагрузкаСервер\Ext\Module.bsl
```

По коду обработчика:

- `Организация` не читается из верхнеуровневого XML-тега документа;
- `Склад` разбирается из XML-узла `Склады/Склад`, но стандартная функция заполнения поля документа не использует это значение напрямую;
- для заполнения полей документа используется таблица соответствий реквизитов из настроек обмена БУС.

Если нужно брать `Организация` и `Склад` с сайта, в настройках заполнения реквизитов загружаемых документов нужно указать:

| Реквизит 1С   | Тип данных                            | Значение      |
| ------------- | ------------------------------------- | ------------- |
| `Организация` | `Сопоставление по свойству документа` | `Организация` |
| `Склад`       | `Сопоставление по свойству документа` | `Склад`       |

При такой настройке обработчик берет значения из `ЗначенияРеквизитов` XML-документа. Для ссылочных реквизитов справочников поиск выполняется по наименованию, поэтому значения с сайта должны буквально совпадать с наименованиями объектов в 1С.

## Что проверить в обработчике 1С

### 1. Найти код загрузки заказов

В конфигураторе/расширении/внешней обработке выполнить поиск по строкам:

- `mode=query`
- `Заказ товара`
- `Документ`
- `ЗначенияРеквизитов`
- `Организация`
- `Склад`
- `ЗаказКлиента`
- `Документы.ЗаказКлиента`

Нужна процедура, которая после получения XML создает или заполняет документ заказа клиента.

### 2. Поставить точки останова

Поставить breakpoint:

1. сразу после чтения XML;
2. перед присвоением полей документа;
3. перед `Записать()` / `Провести()` документа.

На breakpoint проверить:

- видит ли код XML-узел `Организация`;
- видит ли код XML-узел `Склад`;
- читает ли он значения из `ЗначенияРеквизитов`;
- выполняется ли поиск справочников по имени или по GUID;
- чему равны `ДокументОбъект.Организация` и `ДокументОбъект.Склад` перед записью.

### 3. Проверить поиск справочников в 1С

В консоли/отладчике 1С проверить поиск по точным наименованиям из XML:

```bsl
Справочники.Организации.НайтиПоНаименованию("ИП Семерюк Д. В.");
Справочники.Организации.НайтиПоНаименованию("ИП Терещенко Л.В.");

Справочники.Склады.НайтиПоНаименованию("1 СДВ склад");
Справочники.Склады.НайтиПоНаименованию("2 ТЛВ склад");
Справочники.Склады.НайтиПоНаименованию("Intex ОСНОВНОЙ");
```

Если возвращается пустая ссылка, проблема в несовпадении наименований или обработчик должен получать GUID.

## Матрица выводов

| Наблюдение в 1С                                                              | Вывод                                                                     | Действие                                                            |
| ---------------------------------------------------------------------------- | ------------------------------------------------------------------------- | ------------------------------------------------------------------- |
| XML-значения видны, справочники по имени находятся, но поля документа пустые | Обработчик не присваивает `Организация`/`Склад` объекту документа         | Исправить маппинг в обработчике 1С                                  |
| XML-значения видны, но `НайтиПоНаименованию` возвращает пустую ссылку        | В 1С другие точные имена или поиск по имени не используется               | Согласовать точные имена или перейти на GUID                        |
| Обработчик ищет GUID организации/склада                                      | Сайт сейчас передает имена, не GUID организации/склада в документе заказа | Передать нам GUID организаций/складов и ожидаемый XML-фрагмент      |
| Обработчик читает только `ЗначенияРеквизитов`                                | Нужно сверить точные названия реквизитов                                  | Согласовать имена реквизитов: `Организация`, `Склад`, `Соглашение`  |
| Обработчик не заходит внутрь `Контейнер/Документ`                            | Несовместимость структуры CommerceML с обработчиком                       | Согласовать формат: с `Контейнер` или без него, версия `2.09`/`3.1` |

## Что нужно получить от 1С-разработчика

1. Где именно в обработчике заполняются поля `ДокументОбъект.Организация` и `ДокументОбъект.Склад`.
2. По какому источнику они заполняются:
   - верхнеуровневые XML-теги;
   - `ЗначенияРеквизитов`;
   - GUID;
   - настройки обмена/значения по умолчанию.
3. Как выполняется поиск справочников: по `Наименование`, `Код`, `УникальныйИдентификатор`, внешнему ID.
4. Какой XML-фрагмент обработчик ожидает для корректного заполнения организации и склада.
